<details>
<summary>📖 <strong>Salmo do Dia</strong> — Um salmo diferente a cada dia do mês</summary>

<br>

Busca o Salmo correspondente ao dia atual do mês (ex: dia 13 → Salmo 13) via API pública e atualiza o README todo dia ao amanhecer. Uma forma de integrar a leitura dos Salmos à sua rotina de dev.

**Arquivo:** `.github/workflows/salmo.yml`

```yaml
name: Salmo do Dia

on:
  schedule:
    - cron: "0 9 * * *"   # todo dia às 9h (UTC) = 6h horário de Brasília
  workflow_dispatch:

jobs:
  update-psalm:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v3

      - name: Buscar Salmo do dia
        run: |
          DAY=$(date +%-d)
          PSALM=$(curl -s "https://bible-api.com/psalms+${DAY}?translation=almeida" \
            | python3 -c "
          import sys, json
          d = json.load(sys.stdin)
          ref = d['reference']
          text = d['verses'][0]['text'].strip()
          print(f'{ref}:1 — {text}')")
          echo "PSALM=$PSALM" >> $GITHUB_ENV

      - name: Atualizar README.md
        run: |
          sed -i "s|<!-- PSALM -->.*<!-- /PSALM -->|<!-- PSALM -->${{ env.PSALM }}<!-- /PSALM -->|g" README.md

      - uses: stefanzweifel/git-auto-commit-action@v5
        with:
          commit_message: "📖 Salmo do dia atualizado"
```

**Adicione esta linha no `README.md`** onde quiser exibir o salmo:

```html
> 📖 <!-- PSALM -->Carregando salmo...<!-- /PSALM -->
```

> **⚠️ Requisito:** Em *Settings → Actions → General → Workflow permissions*, ative **Read and write permissions**.

</details>

Rotaciona semanalmente uma citação de grandes teólogos e pregadores da história cristã — Spurgeon, Lutero, Calvino, C.S. Lewis, A.W. Tozer e outros — atualizando o README toda segunda-feira.

**Arquivo:** `.github/workflows/teologos.yml`

```yaml
name: Citação de Teólogos

on:
  schedule:
    - cron: "0 6 * * 1"   # toda segunda-feira às 6h (UTC)
  workflow_dispatch:

jobs:
  update-quote:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v3

      - name: Selecionar citação da semana
        run: |
          python3 << 'EOF'
          import datetime, os

          quotes = [
            ("Um grama de fé genuína vale mais que uma tonelada de conhecimento teológico morto.", "C.H. Spurgeon"),
            ("A Bíblia é a autoridade suprema em todos os assuntos que ela trata.", "Martinho Lutero"),

            ("Não é suficiente ter uma opinião de Cristo; precisamos ter Cristo.", "A.W. Tozer"),
            ("Ou Deus é tudo, ou não é nada. Ele deve ser o centro, não apenas um aspecto da vida.", "C.S. Lewis"),
            ("A oração não nos prepara para a grande obra; a oração é a grande obra.", "Oswald Chambers"),
            ("A graça não é simplesmente bondade ou benevolência; é amor que busca sua glória no bem do amado.", "John Murray"),
            ("Sem a cruz, não há cristianismo. Sem ressurreição, não há esperança.", "John Stott"),
            ("A humildade não é pensar menos de si mesmo; é pensar em si mesmo com menos frequência.", "C.S. Lewis"),
            ("Deus nos criou para nós mesmos, e nosso coração vive inquieto até repousar em Ti.", "Agostinho de Hipona"),
            ("A adoração é a resposta da alma redimida ao Deus que a redimiu.", "A.W. Tozer"),
            ("Toda a Escritura é dada por inspiração de Deus e é útil.", "Jerônimo"),
            ("O Evangelismo não é uma opção para a Igreja; é o coração da Igreja.", "Billy Graham"),
          ]

          week = datetime.date.today().isocalendar()[1]
          quote, author = quotes[week % len(quotes)]
          line = f"<!-- QUOTE -->{quote} — *{author}*<!-- /QUOTE -->"

          with open("README.md", "r") as f:
            content = f.read()

          import re
          new_content = re.sub(r"<!-- QUOTE -->.*?<!-- /QUOTE -->", line, content)

          with open("README.md", "w") as f:
            f.write(new_content)

          print(f"Citação da semana {week}: {author}")
          EOF

      - uses: stefanzweifel/git-auto-commit-action@v5
        with:
          commit_message: "✍️ Citação de teólogo atualizada"
```

<summary>⏱️ <strong>WakaTime Stats</strong> — Tempo de código por linguagem</summary>

<br>

Exibe quantas horas você codou por semana, separado por linguagem e projeto, diretamente no README. Atualizado toda segunda-feira.

**Pré-requisitos:**
1. Criar conta em [wakatime.com](https://wakatime.com) e instalar o plugin na sua IDE
2. Gerar uma API Key em *Account Settings → API Key*
3. Adicionar como secret: *Settings → Secrets → Actions* → `WAKATIME_API_KEY`
4. Gerar um token clássico do GitHub com permissão `repo` e adicionar como: `GH_TOKEN`

**Arquivo:** `.github/workflows/wakatime.yml`

```yaml
name: WakaTime Stats

on:
  schedule:
    - cron: "0 0 * * 1"   # toda segunda-feira à meia-noite (UTC)
  workflow_dispatch:

jobs:
  update-readme:
    runs-on: ubuntu-latest

    steps:
      - uses: anmol098/waka-readme-stats@master
        with:
          WAKATIME_API_KEY: ${{ secrets.WAKATIME_API_KEY }}
          GH_TOKEN: ${{ secrets.GH_TOKEN }}
          SHOW_LANGUAGE: "True"
          SHOW_PROJECTS: "True"
          SHOW_EDITORS: "True"
          SHOW_OS: "True"
          SHOW_COMMIT: "True"
          SHOW_TIMEZONE: "True"
          LOCALE: "pt-BR"
```

**Adicione estas marcações no `README.md`** onde os stats devem aparecer:

```markdown
<!--START_SECTION:waka-->
<!--END_SECTION:waka-->
```

<summary>📋 <strong>Atividade Recente</strong> — Últimos eventos públicos do GitHub</summary>

<br>

Lista automaticamente os últimos eventos públicos do seu GitHub (commits, PRs, issues, stars) diretamente no README. Atualizado diariamente.

**Arquivo:** `.github/workflows/activity.yml`

```yaml
name: Atividade Recente

on:
  schedule:
    - cron: "0 0 * * *"   # todo dia à meia-noite (UTC)
  workflow_dispatch:

jobs:
  update-readme:
    runs-on: ubuntu-latest

    steps:
      - uses: jamesgeorge007/github-activity-readme@master
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        with:
          COMMIT_MSG: "⚡ Atividade recente atualizada"
          MAX_LINES: 10
```

**Adicione estas marcações no `README.md`** onde a atividade deve aparecer:

```markdown
<!--START_SECTION:activity-->
<!--END_SECTION:activity-->
```

> **⚠️ Requisito:** Em *Settings → Actions → General → Workflow permissions*, ative **Read and write permissions**.
